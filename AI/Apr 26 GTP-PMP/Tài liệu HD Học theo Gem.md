# 🎯  1. MỤC TIÊU HỆ THỐNG

  

Hệ thống này giúp bạn:

  

- Đọc hiểu câu hỏi PMP dù tiếng Anh không tốt
    
- Nhận diện nhanh context – trigger – cause
    
- Loại đáp án sai theo pattern PMI
    
- Chọn đáp án theo mindset thay vì đoán
    
- Sửa lỗi có hệ thống và tích lũy pattern
    

  

|   |
|---|
|👉  Mục tiêu cuối:<br><br>Nhìn câu hỏi → đoán được hướng đáp án TRƯỚC KHI đọc option|

  

# 🧠  2. CÁCH HỌC CỐT LÕI

  

Bạn sẽ KHÔNG học theo kiểu:

- ❌  Đọc lý thuyết
    
- ❌  Học thuộc
    

  

Mà theo flow:

  

|   |
|---|
|Đọc  →  Hiểu  →  Dự đoán  →  Chọn  →  Sửa  →  Ghi pattern  →  Lặp lại|

  

# 🔁  3. QUY TRÌNH 1 BUỔI LUYỆN (45–60 PHÚT)

  

## 🔹 PHASE 1 — Khởi động (5 phút)

Ôn lại 3 nội dung sau:

- Keyword: FIRST / NEXT / BEST
    
- Elimination rules
    
- 3 pattern từ buổi hôm trước
    

  

## 🔹 PHASE 2 — Luyện chính (30–40 phút)

Mỗi câu làm theo flow 3 bước:

  

✅  Step 1 — Reading (30–40 giây)

Bạn phải ghi ra:

  

|   |   |
|---|---|
|Keyword||
|Context||
|Trigger||
|Cause||
|Compression|(1 câu tiếng Việt ngắn)|
|PMI action|(dự đoán hướng đáp án)|

  

✅  Step 2 — Answer (10–20 giây)

- Loại 2 đáp án sai trước
    
- Sau đó chọn đáp án đúng
    

  

✅  Step 3 — Review (2–3 phút)

- Check lại reading có đúng không
    
- Phân tích cả 4 đáp án
    
- Ghi lỗi nếu sai
    
- Viết pattern rút ra
    

  

|   |
|---|
|⚠️  Đây là phần QUAN TRỌNG NHẤT trong toàn bộ quy trình!|

  

## 🔹 PHASE 3 — Tổng hợp (10–15 phút)

  

1. Đọc lại pattern tích lũy

Ví dụ:

- stakeholder unhappy → engage
    
- team conflict → facilitate
    
- unclear requirement → clarify
    

  

2. Xác định lỗi lặp lại

Ví dụ:

- Hay nhầm trigger
    
- Hay chọn escalate
    
- Hay nhảy bước
    

  

3. Ghi top 3 lỗi cá nhân

  

## 🔹 PHASE 4 — Drill nhanh (5 phút)

- Làm 2–3 câu
    
- Chỉ đọc + đoán hướng
    
- Không cần chọn đáp án cụ thể
    

  

# 🧾  4. CÁCH TRACKING BẰNG FILE EXCEL

  

## 🔹 Sheet 1: Log từng câu

Mỗi dòng = 1 câu. Điền đủ các cột:

  

|         |         |         |       |             |             |     |            |         |
| ------- | ------- | ------- | ----- | ----------- | ----------- | --- | ---------- | ------- |
| Keyword | Context | Trigger | Cause | Đáp án chọn | Đáp án đúng | ✓/✗ | Error type | Pattern |
| ...     | ...     | ...     | ...   | ...         | ...         | ... | ...        | ...     |

  

## 🔹 Sheet 2: Error Summary

Tổng hợp lỗi để thấy mình sai kiểu gì nhiều nhất:

  

|   |   |
|---|---|
|Error Type|Count|
|Trigger error|8|
|Elimination error|5|
|Sequence error|3|

  

## 🔹 Sheet 3: Pattern Library

Đây là tài sản quan trọng nhất. Mỗi pattern = 1 dòng:

  

|   |
|---|
|Pattern (Tình huống → Hành động PMI)|
|stakeholder unhappy → engage|
|team frustrated → discuss|
|change request → assess impact|
|unclear requirements → clarify|
|risk identified → document & plan response|
